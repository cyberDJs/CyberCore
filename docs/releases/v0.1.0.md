# CyberCore v0.1.0

Release date: **2026-07-20**

CyberCore v0.1.0 is the first foundation release of the public framework.

It establishes CyberCore as an **Infrastructure Context & Intelligence Platform**
with an **AI-first Infrastructure Control Plane** as the technical core.

## What is included

### Foundation

- CyberCore Identity v1.0.
- Governance, manifesto, glossary, and design-system foundation.
- Foundation documents for engineering method, terminology, decision model, and knowledge model.
- Public Framework + Private Overlay boundary.
- Architecture Decision Records for the control-plane direction, identity freeze, and CXP artifact format.

### Runtime baseline

```bash
cybercore doctor
cybercore status
cybercore sync
cybercore verify /path/to/WB-XXXX
cybercore --repo /path/to/CyberCore apply --dry-run /path/to/WB-XXXX
```

The runtime provides a unified CLI, read-only verification, explicit approval
before apply, repository safety checks, structured Work Block validation, and
lifecycle events.

### CXP artifact system

```bash
cybercore build ./source \
  --output ./dist \
  --artifact-id WB-0010-example \
  --title "Example Artifact" \
  --created-at 2026-07-20T09:00:00Z
```

CXP/1 artifacts are deterministic, content-addressed, compressed with Zstandard,
and identified by SHA-256.

## Public-release boundary

The public repository contains framework code, specifications, schemas, tests,
sanitized examples, and documentation.

Private overlays remain responsible for credentials, production inventory,
customer data, runtime state, private topology, telemetry, and environment
configuration.

## Trust model

v0.1.0 provides deterministic integrity verification. It does not yet provide
cryptographic publisher identity or encrypted transport.

Planned follow-up work:

- Ed25519 publisher signatures;
- age/X25519 encrypted envelopes;
- CXP Runtime reader;
- registry and transport backends;
- provider integrations;
- Git workflow automation.

This is a **foundation release**. It is suitable for public review,
specification alignment, and runtime baseline development. Production-changing
automation remains explicitly human-controlled.
