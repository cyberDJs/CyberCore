# CyberCore v0.1.0-alpha.1

Release date: **2026-07-16**

CyberCore is an **Infrastructure Context & Intelligence Platform** and an
**AI-first Infrastructure Control Plane**.

This first alpha checkpoint establishes the project constitution, executable
runtime, controlled Work Block boundary, and deterministic CXP artifact builder.

## Highlights

### Foundation

- Identity v1.0 and governance.
- Public Framework + Private Overlay.
- Explicit engineering and decision model.
- Architecture, glossary, manifesto, and design direction.

### Runtime Alpha

```bash
cybercore doctor
cybercore status
cybercore sync
cybercore verify /path/to/WB-XXXX
cybercore --repo /path/to/CyberCore apply --dry-run /path/to/WB-XXXX
```

The Runtime validates Work Blocks, enforces repository safety gates, requires
explicit approval before mutation, and supports lifecycle hooks.

### CXP Artifact System

```bash
cybercore build ./source \
  --output ./dist \
  --artifact-id WB-0010-example \
  --title "Example Artifact" \
  --created-at 2026-07-16T09:00:00Z
```

CXP/1 artifacts are deterministic, content-addressed, compressed with
Zstandard, and identified by SHA-256.

## Installation for development

```bash
git clone https://github.com/cyberDJs/CyberCore.git
cd CyberCore

python3 -m venv .venv
source .venv/bin/activate

python -m pip install -e '.[dev]'
pytest
cybercore --repo "$PWD" doctor
```

## Trust model

This alpha provides deterministic integrity verification. It does not yet
provide cryptographic publisher identity or encrypted transport.

Planned:

- Ed25519 signatures;
- age/X25519 encrypted envelopes;
- CXP Runtime reader;
- registry and transport backends;
- Git workflow automation.

## Release status

This is an **alpha development release**. Interfaces and artifact details may
still evolve before beta, but changes must follow Governance and ADR review.
